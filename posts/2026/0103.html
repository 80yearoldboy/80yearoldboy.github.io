类欧几里得算法学习笔记1（f函数篇）

Update on 1.11 修复了一些LaTeX的语法错误。

  之前打某场比赛时看到这样一道题：
已知某一次函数的表达式为 $$ \displaystyle y=\frac{ax+b}{c} $$，在 $$x \in [0,n] $$ 且 $$x \in \mathbb Z $$ 时，求在函数图像下有多少个整点。

  当时还不知道这是个什么东西，但后来发现这是类欧几里得的板子。
  我们再来观察一下刚才的问题，发现该问题的答案可以用一下式子表示：

∑i=0n⌊ai+bc⌋ 

  但是直接算的话肯定会超时，但类欧几里得算法提供了计算该式子的一种更快的方式。
  接下来是推导过程。
  我们令 f(a,b,c,n)=∑i=0n⌊ai+bc⌋。
  当 c≤a 或 c≤b 时：

f(a,b,c,n)=f(a%c,b%c,c,n)+n(n+1)2⋅⌊ac⌋+(n+1)⋅⌊bc⌋

否则令 m=an+bc，则可做如下变形：

f(a,b,c,n)=∑i=0n∑j=1m[ai+bc≥j]=∑i=0n∑j=0m−1[ai+bc≥j+1]=∑i=0n∑j=0m−1[ai≥cj+c−b]=∑i=0n∑j=0m−1[ai>cj+c−b−1]=∑i=0n∑j=0m−1[i>cj+c−b−1a]=∑j=0m−1∑i=0n[i>cj+c−b−1a]=∑j=0m−1(n−cj+c−b−1a)=∑j=0m−1n−∑j=0m−1cj+c−b−1a=∑j=1mn−∑j=0m−1cj+c−b−1a=mn−f(c,c−b−1,a,m−1)

  特别的，当 a=0 时，f(a,b,c,n)=(n+1)⋅⌊bc⌋。
  这也是整个递归的结束条件。   这种算法可达到 O(log n) 的时间复杂度。
  下面给出代码。

cpp
#include<bits/stdc++.h>
using namespace std;
int a,b,c,n;
int f(int a,int b,int c,int n){
	if(a==0){
		return (n+1)*(b/c);
	}
	if(a>=c||b>=c){
		return f(a%c,b%c,c,n)+n*(n+1)/2*(a/c)+(n+1)*(b/c);
	}
	else{
		int m=(a*n+b)/c;
		if(m==0)return 0;
		return m*n-f(c,c-b-1,a,m-1);
	}
}
int main(){
	cin>>a>>b>>c>>n;
	cout<<f(a,b,c,n);
	return 0;
}

  同时，类欧几里得还有 g(a,b,c,n)=∑i=0ni⌊ai+bc⌋ 和 h(a,b,c,n)=∑i=0n⌊ai+bc⌋2。
